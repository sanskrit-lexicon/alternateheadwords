## Project Overview

**alternateheadwords** identifies and generates alternate headword spellings for all Cologne dictionaries — entries where the headword contains a parenthesized variant (e.g., `kube(ve)raH` → `kuveraH` or `kubeveH`). These alternates improve search coverage in the CDSL API.

## Architecture

| File/Directory | Purpose |
|---|---|
| `redo.sh` | Main orchestration script: runs the full pipeline for all dictionaries |
| `scripts/` | Core Python pipeline scripts |
| `scripts/ahw.py` | Extracts entries with parenthesized variants from `hw0.txt` → `ahw0.txt` |
| `scripts/ahw1.py` | Generates candidate alternate headwords and scores them → `ahw1.txt` |
| `scripts/embedded.py` | Handles embedded-parenthesis patterns |
| `scripts/levenshtein.py` | Levenshtein distance for scoring alternate candidates |
| `scripts/listngrams.py` | N-gram analysis for abnormal bigram/trigram detection |
| `scripts/suggest.py` | Suggests corrections for flagged alternates |
| `scripts/compare_pwg_pw.py` | Cross-dictionary comparison between PW/PWG headword alternates |
| `scripts/transcoder.py` | SLP1 transcoding utilities |
| `data/` | Per-dictionary output files (`<dict>ahw0.txt` through `<dict>ahw3.txt`) |

### Pipeline (per dictionary)

1. **ahw0.txt** — lines from `<dict>hw0.txt` that contain `( )` parenthesized variants (format: `lnum,pc:headword:L1,L2`)
2. **ahw1.txt** — adds suggested alternates (format: `code:headword:preferred:scrap`)
   - code 3: clear correction; code 0: ambiguous
3. **ahw2.txt** — reverts suggestions with abnormal n-grams back to code 0
4. **ahw3.txt** — manually reviewed copy of ahw2.txt (copy ahw2 → ahw3, then edit)

### Input

Uses `<DICT>hw0.txt` files from the Cologne server/XAMPP installation (generated by `csl-pywork`).
